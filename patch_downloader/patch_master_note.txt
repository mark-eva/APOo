<a name="aref_section11"></a><h2 class="km">1.0 OVERVIEW</h2>
<p>Oracle provides Database patches for both proactive and reactive maintenance:</p>
<ul type="disc">
<li>Reactive Patches:</li>
<ul type="circle">
<li>are usually delivered as &ldquo;Interim Patches&rdquo;</li>
<li>were historically known as &ldquo;one-off&rdquo; patches</li>
<li>are provided on demand for a given &ldquo;defect, version, platform&rdquo; combination</li>
<li>go through basic sanity tests</li>
<li>fixes will usually be included in the next relevant Patch Set Release</li>
</ul>
</ul>
<ul type="disc">
<li>Proactive Patches</li>
<ul type="circle">
<li>address high impact bugs that affect a given configuration</li>
<li>contain proven low risk fixes</li>
<li>go through extra levels of testing, determined by the feature(s) affected</li>
<li>are available on "My Oracle Support" by clicking on the "Patches &amp; Updates" tab</li>
</ul>
</ul>
<p>For more information on Database Patch Delivery Methods see "Oracle Database - Overview of Database Patch Delivery Methods - 12.1.0.2 and older", <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=1962125.1'>Note 1962125.1</a> and "Oracle Database - Overview of Database Patch Delivery Methods for 12.2.0.1 and greater", <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=2337415.1'>Note 2337415.1</a> <br /> Oracle releases multiple types of Database patches for the Database and Clusterware installations:</p>
<ol>
<li>Quarterly Release Updates (Updates) and Quarterly Release Update Revisions (Revisions) - Applies to Database 12.2.0.1 and later</li>
<li>Quarterly Patch Set Updates (PSUs) and Quarterly Proactive Bundle Patches (BPs) - Applies to Database 12.1.0.2 and prior</li>
<li>One-off Patches - Applies to all Database versions</li>
</ol>
<p>The Database PSU is available for Database installations and the Grid Infrastructure (GI) PSU / Database Proactive BP for Clusterware installations for Database 12.1.0.2 and prior. Starting with Database 12.2.0.1, the Quarterly Release Updates (Updates) will replace Bundle patches and Quarterly Release Update Revisions (Revisions) will replace the PSU's to improve the quality and experience of proactive maintenance. <br /> For more information on Database Release Update (Update) and Release Update Revision (Revision) see My Oracle Support <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=2285040.1'>Note 2285040.1</a> Release Update Introduction and FAQ.</p>
<p>These patches are released quarterly (Jan, Apr, Jul, Oct). There may be Monthly updates released in the first few months of a new release to accelerate stabilization.</p>
<p>Customers can get standalone one-off patches and diagnostic patches as Oracle has delivered for many years for Update, Revision, PSU or BP. Emergency patches will be delivered on request on top of any Updates or Revisions or PSUs or BPs for a supported release as long as technically feasible. If a fix has severe impact or issue is encountered by many customers, then fix will be considered for inclusion into the various bundles. Customers should update to the latest bundle that includes their emergency fix once it is available.</p>
<p>&nbsp;</p>

<a name="aref_section12"></a><h3 class="km"><a name="mozTocId773320" class="mozTocH3"></a><span class="secnum">1.1</span> Oracle Database Patching - SPU vs PSU/BP</h3>
The Database Security Patch Updates (SPU) and Patch Set Updates (PSU) / Bundle Patches (BP) that are released each quarter contain the same security fixes. However, they use different patching mechanisms, and PSU/BP include both security and critical fixes.<br /> <br /> For more information on Database Patch Delivery Methods see My Oracle Support <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=1962125.1'>Note 1962125.1</a>, Oracle Database - Overview of Database Patch Delivery Methods - 12.1.0.2 and older.
<p style="font-weight: bold;">NOTE: Applying a SPU on an installation with an installed PSU/BP is not supported</p>
</div>
<p>&nbsp;</p>
<p>&nbsp;</p>
<div class="sect2">
<a name="aref_section13"></a><h3 class="km"><a name="mozTocId514688" class="mozTocH3"></a>1.2 OJVM PSU can be installed in "Conditional Rolling Install" manner in certain use cases</h3>
<p>Starting in January 2017, the OJVM PSU for 12.1.0.2 and 11.2.0.4 may be installed in a "Conditional Rolling Install" fashion for the following use cases:<br /> <br /> &nbsp;&nbsp;* No OJVM usage<br /> &nbsp;&nbsp;* OJVM used by non-critical jobs and programs<br /> &nbsp;&nbsp;* OJVM used by critical functions isolated as services<br /> &nbsp;&nbsp;* OJVM used extensively, not isolated, and downtime is tolerated<br /> &nbsp;&nbsp;* OJVM used by critical functions and minimal downtime is required<br /> <br /> See My Oracle Support <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=2217053.1'>Note 2217053.1</a> for more details.</p>
</div>
<p>&nbsp;</p>
<p>&nbsp;</p>

<a name="aref_section14"></a><h3 class="km"><a name="mozTocId267914" class="mozTocH3"></a><span class="secnum">1.3</span> Proactive Replacement Interim Patches for Conflicts with Database Proactive Patches</h3>
<p>So that customers can quickly consume new quarterly Database Proactive Patches, Oracle determines any existing interim patches that conflict with the new proactive patches, and generates replacement interim patches as needed.</p>
<p>For more information, see My Oracle Support <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=1998563.1'>Note 1998563.1</a> Proactive Replacement Interim Patches for Conflicts with Database Proactive Patches.</p>

<a name="aref_section16"></a><h2 class="km">2.0 What's New</h2>
<a name="aref_section17"></a><h3 class="km">2.1 Bug fixes that may change an existing Optimizer execution plan</h3>
<p>Beginning with database version 12.2.0.1 and beyond, database Bug fixes that may change an existing Optimizer execution plan will be delivered with the Bug fix disabled by default. Such fixes will have to be explicitly enabled after the patch has been applied. Details on this, including the commands to explicitly enable such Bug fixes are presented in MOS <a href='/epmos/faces/DocumentDisplay?parent=DOCUMENT&sourceId=756671.1&id=2147007.1'>Note 2147007.1</a>.</p>
